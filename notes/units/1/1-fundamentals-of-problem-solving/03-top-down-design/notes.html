Top down design

Top down design breaks down a problem into its smallest constituent pieces, so you can see what the problem entails, and what each problem in that problem entails, and so on.

e.g.

                 _________
                |         |
                | Problem |
                |_________|
               /     |     \
              /      |      \
     _______ /    ___|___    \ ________
    |       |    |       |    |        |
    | Input |    | Logic |    | Output |
    |_______|    |_______|    |________|
