Fetch-Execute cycle

Note: see /exams/comp2/common-questions.md for a definitive guide on questions about the FET cycle.

  1. Instruction time / I-time

    1. Fetch: Next instruction fetched from address currently in PC and stored in IR. Finally, increment PC.
    2. Fetch address <- PC
    3. Store address -> IR
    4. Increment PC
    5. Decode:

  3. Load PC -> MAR

  4. Increment PC

  5. Execute instruction

  6. PC -> MAR to determine location of next instruction

    1. CU fetches instruction from memory & puts in IR
    2. CU decodes instruction & directs that the necessary data is moved from memory -> ALU.
      • e.g. if instruction is reg. 1 + reg. 2, then reg. 1 & 2 must be fetched

  7. ALU performs an operation

  8. ALU stores result in a register

Register notation

The following is adapted from the AQA Computing Wikibook.

  1. \(MAR \gets [PC]\)
  2. \(MBR \gets [Memory]_{MAR\ address}\) and, simultaneously, \(PC \gets [PC]+1\)
  3. \(CIR \gets [MBR]\)
  4. \([CIR]\) decoded then executed
